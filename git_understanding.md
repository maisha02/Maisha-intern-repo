Difference between staging and committing
Staging means choosing the file we want to commit next. And for that we use the `git add` command.
Committing means saving those staged changes to the project history. This is done using the `git commit` command.

Why Git separates staging and committing
Git keeps these steps separate so that we can choose exactly which change should be saved together. This helps keep commits organized.

When staging without committing is useful
Staging without committing can be useful when we want to check our changes before commiting them. It also helps us when we want to commit some changes now and leave few for later.
