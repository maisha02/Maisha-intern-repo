# Static Analysis Checks in CI/CD (#57)

## What is the purpose of CI/CD?
CI/CD helps to test and manage code changes in an automatic way. It makes sure code is checked before it is merged which helps us to ensure no mistake is made.

## How does automating style checks improve project quality?
Automating style checks improves project quality by finding problems early. It keeps the file consistent, easier to read, and easier to maintain.

## What are some challenges with enforcing checks in CI/CD?
One challenge is that checks can fail for small issues, which can lead to a slow down development. Another challenge is setting up the tools correctly so they are useful and not too strict.

## How do CI/CD pipelines differ between small projects and large teams?
Small projects usually have simpler pipelines with a few checks. Large teams often use more complex pipelines with many tests, reviews, and deployment steps.
